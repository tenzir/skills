# Readability Lens

Use this lens when code clarity, naming, or local comprehension is the main
concern.

Focus on:

- naming quality and intent revelation
- language-idiomatic patterns
- control-flow clarity
- comments that are missing, misleading, or redundant
- expressions or helpers that make simple logic hard to follow

Prefer issues that meaningfully affect comprehension. Ignore style nits already
covered by formatters and linters.
