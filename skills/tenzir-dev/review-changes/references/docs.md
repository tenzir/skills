# Documentation Lens

Use this lens when code changes affect docs, docstrings, examples, or README
content.

Focus on:

- Accuracy: docs match the actual behavior
- Completeness: new behavior is documented where users will look for it
- Example validity: examples still run and show the right output
- Cross-links and references: linked pages and related docs stay coherent
- Style: wording follows Tenzir technical-writing guidance

For code changes, check whether a user-facing change should update `.docs/` or
other nearby documentation.
