# User Experience Lens

Use this lens when the diff changes user-facing behavior, CLI output, error
messages, setup flows, or defaults.

Focus on:

- clarity of messages and prompts
- discoverability of new or changed behavior
- consistency with existing terminology
- user feedback during long-running or risky operations
- sensible defaults for the common path

Prefer concrete findings that affect how a user understands or recovers from
the feature.
